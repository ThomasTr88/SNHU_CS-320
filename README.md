# CS 320 Portfolio Reflection

## How can I ensure that my code, program, or software is functional and secure?

Throughout this course, I learned that writing unit tests is one of the best ways to ensure software is functional and secure. For Project One, I used JUnit tests to verify that each class met the project requirements. I tested both valid and invalid inputs, including null values, duplicate IDs, empty strings, invalid phone numbers, and past dates. Testing these scenarios helped me identify problems early and ensured that the program behaved correctly. I also learned that following validation rules and reviewing code coverage helps improve software quality and reliability.

## How do I interpret user needs and incorporate them into a program?

I learned that understanding the project requirements before writing code is very important. Before I started each service, I carefully reviewed the requirements and created tests that matched each one. For example, I verified that contact IDs could not be changed, phone numbers contained exactly ten digits, and appointment dates could not be in the past. I also reviewed feedback from previous milestones and updated my code to better meet the customer's requirements. This helped me create software that matched what the user expected.

## How do I approach designing software?

I now approach software design by first understanding the requirements, then planning the classes and methods before writing code. After implementing the code, I create unit tests to verify that each requirement has been met. During this course, I also learned the importance of making code easier to maintain by reducing duplicate code and using reusable setup methods such as @BeforeEach. Testing throughout development helped me improve the quality of my code and increased my confidence that the software worked correctly.
